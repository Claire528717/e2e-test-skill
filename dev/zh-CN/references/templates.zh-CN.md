# PRD E2E 测试代理模板

## 专用测试文件

整条测试生命周期只使用一份专用 Markdown 文件。推荐文件名格式如下：

`e2e-test-plan-{feature}-{run_id}.md`

如果用户没有指定路径，默认放在 `e2e-test-plans/` 下，证据放在 `e2e-test-evidence/{run_id}/` 下。

在执行前，这个文件是 E2E 测试计划。只有在用户明确授权执行并且运行开始后，才把执行结果、BUG 和清理状态追加到同一份文件里。

```markdown

- 必须由用户提供的数据：

UI 保真要求：
- 每个被测用户动作都必须从真实用户也会使用的可见入口开始。
- 浏览器自动化可以点击、输入、选择、拖拽、确认和等待 UI，但不能通过修改隐藏 DOM、调用组件内部方法、派发合成产品事件、写入 storage/session，或调用后端 API 来替代 UI 操作。
- 如果没有可见的用户路径可走，就把用例标记为阻塞或失败。

UI 操作路径：
1. 打开 {产品/页面}。
2. 以 {参与者} 登录。
3. 点击 {可见标签}。
4. 打开 {页面/抽屉/弹窗/标签页}。

7. 点击 {按钮}。
8. 确认可见结果：{toast/状态/列表行/详情文本}。

独立验证：

4. 验证 {外部对象/内容/数量/版本}。
5. 交叉核对 {产品引用} 与 {外部引用} 一致。

预期结果：
- UI：
- 产品数据库：
- 外部系统：
- 副作用：

需采集的证据：
- 截图或可见状态：
```
